Intro

The best way to understand Subledger is to see it in action.

Imagine you’re building a pay-per-use newspaper reader called Payper (check it out at Payper.io). Payper will enable users to read articles from any publication for 0.10c per page.

Here are the transactions that Payper needs to handle:

Re-charge: $10 gets charged to the customer's credit card and is added to their Payper account balance.

Read: Customer reads a page, 0.10c gets deducted from their account balance, 0.08c gets recorded as payable to the publisher (e.g New York Times) and 0.02c gets added to Payper's revenue for the month.

We can use payment APIs like Balanced, Stripe and BrainTree to charge the user’s credit card and payout to publishers. For everything else we can use Subledger.

Now let's get Payper set up in Subledger!

These docs use generic IDs that will work, but we suggest creating your own IDs and using them instead.

You can also check out a complete listing of our API endpoints here.

Authentication

Create an Identity and store your ‘key_id’ and ‘secret’ for future calls.

Throughout these docs we refer to 'active_*_id' as '*_id'. For example 'active_key_id' is referred to as 'key_id' 

EXAMPLE REQUEST
curl -X POST -H "Content-Type: application/json" -d \
'{
		"email": "me@payper.io",
		"description": "payer",
		"reference": "http://payper.io"
}' \
https://api.subledger.com/v1/identities

EXAMPLE RESPONSE
{
		"active_identity": {
				"id": "VEt7WZdKDR0yf1X72MLX81",
				"email": "me@payper.io",
				"description": "payer",
				"reference": "http://payper.io",
				"version": 1
		},
		"active_key": {
				"id": "0sdsdr2dvCTu1ZKneQBDyl",
				"identity": "VEt7WZdKDR0yf1X72MLX81",
				"secret": "ZJmnsty6mzBZEyvViXpbW3"
		}
}

Create Account Categories

Categories are used to group accounts for reporting.

Report Structure

In this example we are going to put accounts into the categories and subcategories required to render this basic report for the Payper business:

Assets (category)

  • Cash (subcategory)
  • Cash (account)

Liabilities (category)

  • AP (subcategory)
  • AP/Publisher (account)
  • Defered Revenue (subcategory)
  • Defered Revenue/User (account)

Now let’s create the categories and subcategories. 

DEFINITION 
curl -X POST -H "Content-Type: application/json" -d \
'{
		"description": "Assets",
		"reference": "http://payper.io/assets",
		"normal_balance": "debit"
}' \
-u key_id:secret 
https://api.subledger.com:443/v1/orgs/org_id/books/book_id/categories

Create Accounts

Accounts will be used to record the location/state of money in your application.

We will create accounts and attach them to categories as specified in the report structure we laid out above.

Global accounts

The Payper business has 2 global accounts: ‘Cash’ and ‘Revenue’

Most applications have a few ‘one off’ global accounts and accounts for each buyer/seller in the application.

Post Journal Entries

Money movement between accounts is recorded by posting a double-entry journal entry. The 'double' entry records which accounts the money moved from and to.

Subledger requires time and date in ISO_8601 format

Re-Charge Journal Entry

Every time a user adds $10 to their account balance we need to post this journal entry:

  • Debit [10.00]
  • Credit DeferredRevenue(john@smith.com)[10.00] 
DEFINITION
	 curl -X POST -H "Content-Type: application/json" -d \
	 '{
		 "effective_at": "Time-Date",
		 "description": "Re-Charge(john@smith.com",
		 "reference": "http://payper.io/payment_receipt",
		 "lines": [
			 {
				 "account": "account_id",
				 "value": {
					 "type": "debit",
					 "amount": "10.00"
				 }
			 },
			 {
				 "account": "account_id",
				 "value": {
					 "type": "credit",
					 "amount": "10.00"
				 }
			 }
		 ]
	 }' \
	 -u key_id:secret https://api.subledger.com/v1/orgs/org_id/books/book_id/journal_entries/create_and_post
					
EXAMPLE REQUEST
	 curl -X POST -H "Content-Type: application/json" -d \
	 '{
		 "effective_at": "2013-09-29T10:40Z",
		 "description": "Re-Charge(john@smith.com)",
		 "reference": "http://payper.io/payment_receipt",
		 "lines": [
			 {
				 "account": "Z8KjRhQJy0LAGlBObU6cq1",
				 "value": {
					 "type": "debit",
					 "amount": "10.00"
				 }
			 },
			 {
				 "account": "0sjv5Zr7VG5DzMSHruOe28",
				 "value": {
					 "type": "credit",
					 "amount": "10.00"
				 }
			 }
		 ]
	 }' \
	 -u 0sdsdr2dvCTu1ZKneQBDyl:ZJmnsty6mzBZEyvViXpbW3 https://api.subledger.com/v1/orgs/WQQF5ppPt62EAMmPnNkx42/books/MKscbFJDlyd8dX6gAaTNv7/journal_entries/create_and_post
					
EXAMPLE RESPONSE
	 {
			 "posting_journal_entry": {
					 "id": "PBa462iMYV6VrjaGiJINx6",

Setup Global Reporting

Global Reporting aggregates account balances into category balances and provides a financial snapshot of all commerce across your application. This is useful for internal reports and answering the question of ‘are we making any money yet?’

Create report 

DEFINITION
curl -X POST -H "Content-Type: application/json" -d \
'{
		"description": "My First Report",
		"reference": "http://happy.com"
}' \
-u key_id:secret https://api.subledger.com:443/v1/orgs/org_id/books/book_id/reports
