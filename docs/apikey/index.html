Create an API Key

An API key is created when an identity is created in Subledger. The response contains both an active_identity and an active_key with your credentials, so you can securely interact with Subledger. 

curl -X POST -H "Content-Type: application/json" -d \
"email":          "demo@subledger.com",
"description":    "lemonade demo identity",
"reference":      "http://sublegder.com"
}' \
https://api.subledger.com/v2/identities

and the response received should look like this: 

{
"active_identity":  {
"id":             "Qse2119xCatsL3UuUIFWCp",
"email":          "demo@subledger.com",
"description":    "lemonade demo identity",
"reference":      "http://sublegder.com",
"version":        1
},
"active_key":     {
"id":             "Z0NpheNlvXLMJCoj0612hr",
"identity":       "Qse2119xCatsL3UuUIFWCp",
"secret":         "rr54CeYEOAZT5KlyUJ9sP7"
}
}

We recommend storing this entire response securely for future reference. It is possible to create additional keys and archive active keys to disable them. This allows for key rotations to be performed where multiple keys are active during the rotation.
